# 操作日志（界面日志区 + 操作 TXT）

## 实现说明（当前代码行为）

**统一策略**：`MainWindow::appendUiActionLog(category, message)` **始终**先调用 `appendLogLine(QString("[%1] %2").arg(category, message))`，再视「操作 TXT 开关 + 路径」追加写入磁盘。因此 **分类操作日志与界面右下角日志区已对齐**；开启操作 TXT 时，同一条内容会出现在界面并落盘。

**仍仅用 `appendLogLine`、不经 `appendUiActionLog` 的内容**（高频或纯技术流，避免操作 TXT 暴涨）：

- `[ALG]` 在线推理/算法结果（逐条）
- `[ACQ][INFO]` / `[ACQ][WARN]` 采集线程状态
- `[CSV][INFO]` / `[CSV][ERROR]` CSV 工作线程
- `[UI] 已清空日志显示区` 通过 `appendUiActionLog("UI", …)` 写入，会进 TXT（若开启）

**未单独打日志（刻意省略）**：曲线拖拽/缩放导致 `m_chartAutoFollow = false`（频率过高）；`main.cpp` 内无 `MainWindow` 日志（启动摘要已在构造与 `initThreads` 中覆盖）。

## 已实现条目与代码位置（`mainwindow.cpp`）

| 分类 | 说明 |
|------|------|
| **INIT** | 构造结束：主界面与 30ms 绘图定时器；`initThreads` 结束：三线程已 start |
| **SERIAL** | 构造后摘要（默认端口/波特）；打开/取消/参数无效/确定后完整参数 |
| **NETWORK** | `initThreads` 内加载并应用后的摘要；打开对话框、取消、确定后完整配置 |
| **UI** | 开始失败/忽略/取消/开始采集摘要；停止（区分是否在采集）；保存确定/取消及四路开关+路径；保存对话框内新建路径、打开文件夹、打开串口；图表对话框打开/取消/切换模式；清除会话取消；清空日志区；暂停/恢复自动滚动；打开 EEG_CSV 按钮 |
| **STREAM / 其它 category** | 流缺口、链路诊断：经 `appendUiActionLog`，界面与 TXT 一致 |
| **LOG** | LogBuffer 丢弃告警：经 `appendUiActionLog` |
| **STAT / SYS** | 测试链路报表与测试消息：经 `appendUiActionLog`（不再重复 `appendLogLine`） |
| **SESSION** | `restartSessionWithReset` 执行摘要 |
| **PLOT** | 双击图表恢复 X 轴自动跟随 |
