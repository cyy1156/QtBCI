# 更换图表三选一（原始/FFT/频段功率）缓存绘图方案

目标：点击“更换图表”按钮后，弹出三个单选项（只能选一个）：

1. 原始数据图像（时域）
2. FFT 频谱图像
3. 频率功率频段图像（delta/theta/alpha/beta/gamma）

并统一采用“先入缓存，再由 UI 线程定时器取缓存绘图”的模式，避免跨线程直接重绘造成卡顿。

---

## 0) 先确认当前工程状态（你问的重点）

基于你当前代码：

- 已有缓存：`m_plotCache`（`MainWindow`）  
  来源：`AlgorithmEngine::plotChunkReady` -> UI lambda -> `m_plotCache += pc.y`
- 未缓存（仅写 CSV）：`fftResultReady`、`spectrumReady`  
  目前连接到 `CsvLogWorker::onFftResult/onSpectrumResult`，没有放进 UI 绘图缓存。

所以你的判断是对的：后两种图现在还没有 UI 绘图缓存。

---

## 1) 设计：三种图统一“缓存->UI绘图”

建议在 `MainWindow` 增加图表模式枚举与缓存：

```cpp
enum class ChartMode {
    RawTime = 0,      // 原始/预处理时域
    FftSpectrum = 1,  // FFT 五频段幅值趋势（或频谱线）
    BandPower = 2     // 频段功率趋势（delta/theta/alpha/beta/gamma）
};

ChartMode m_chartMode = ChartMode::RawTime;

// 原始时域缓存（你已有）
QVector<double> m_plotCache;

// FFT 缓存：每个窗口一组五频段值
struct FftBandPoint {
    double delta = 0.0;
    double theta = 0.0;
    double alpha = 0.0;
    double beta = 0.0;
    double gamma = 0.0;
    quint64 seqEnd = 0;
};
QVector<FftBandPoint> m_fftCache;

// 功率频段缓存：每个窗口一组五频段值
struct PsdBandPoint {
    double delta = 0.0;
    double theta = 0.0;
    double alpha = 0.0;
    double beta = 0.0;
    double gamma = 0.0;
    quint64 seqEnd = 0;
};
QVector<PsdBandPoint> m_psdCache;

int m_featureCacheMax = 512; // FFT/PSD 每种缓存最大窗口数
```

---

## 2) “更换图表”按钮：三选一（只能选一个）

假设你新增按钮名 `pushButton_changeChart`，槽函数示例：

```cpp
void MainWindow::on_pushButton_changeChart_clicked()
{
    QDialog dlg(this);
    dlg.setWindowTitle(QStringLiteral("更换图表"));

    auto *lay = new QVBoxLayout(&dlg);
    auto *rRaw = new QRadioButton(QStringLiteral("原始数据图像"), &dlg);
    auto *rFft = new QRadioButton(QStringLiteral("FFT 频谱图像"), &dlg);
    auto *rPsd = new QRadioButton(QStringLiteral("频率功率频段图像"), &dlg);

    auto *group = new QButtonGroup(&dlg);
    group->setExclusive(true); // 只能选一个
    group->addButton(rRaw, static_cast<int>(ChartMode::RawTime));
    group->addButton(rFft, static_cast<int>(ChartMode::FftSpectrum));
    group->addButton(rPsd, static_cast<int>(ChartMode::BandPower));

    switch (m_chartMode) {
    case ChartMode::RawTime: rRaw->setChecked(true); break;
    case ChartMode::FftSpectrum: rFft->setChecked(true); break;
    case ChartMode::BandPower: rPsd->setChecked(true); break;
    }

    lay->addWidget(rRaw);
    lay->addWidget(rFft);
    lay->addWidget(rPsd);

    auto *box = new QDialogButtonBox(QDialogButtonBox::Ok | QDialogButtonBox::Cancel, &dlg);
    connect(box, &QDialogButtonBox::accepted, &dlg, &QDialog::accept);
    connect(box, &QDialogButtonBox::rejected, &dlg, &QDialog::reject);
    lay->addWidget(box);

    if (dlg.exec() != QDialog::Accepted)
        return;

    m_chartMode = static_cast<ChartMode>(group->checkedId());
    m_plotDirty = true; // 下个 UI tick 立即按新模式重绘
}
```

---

## 3) 把 FFT / PSD 结果放入各自缓存

在 `initThreads()` 新增两条连接（UI 线程 lambda 中入缓存）：

```cpp
connect(m_alg, &AlgorithmEngine::fftResultReady, this, [this](const FftResult &fr){
    FftBandPoint p;
    p.delta = fr.delta;
    p.theta = fr.theta;
    p.alpha = fr.alpha;
    p.beta  = fr.beta;
    p.gamma = fr.gamma;
    p.seqEnd = fr.seqEnd;
    m_fftCache.push_back(p);
    if (m_fftCache.size() > m_featureCacheMax)
        m_fftCache.remove(0, m_fftCache.size() - m_featureCacheMax);
    m_plotDirty = true;
});

connect(m_alg, &AlgorithmEngine::spectrumReady, this, [this](const SpectrumResult &sp){
    PsdBandPoint p;
    p.delta = sp.delta;
    p.theta = sp.theta;
    p.alpha = sp.alpha;
    p.beta  = sp.beta;
    p.gamma = sp.gamma;
    p.seqEnd = sp.seqEnd;
    m_psdCache.push_back(p);
    if (m_psdCache.size() > m_featureCacheMax)
        m_psdCache.remove(0, m_psdCache.size() - m_featureCacheMax);
    m_plotDirty = true;
});
```

## 4) UI 线程定时器按模式绘图

在 `onPlotRefreshTick()` 中按 `m_chartMode` 分发：

```cpp
void MainWindow::onPlotRefreshTick()
{
    if (!m_plotDirty || !m_customPlot)
        return;
    m_plotDirty = false;

    switch (m_chartMode) {
    case ChartMode::RawTime:
        drawRawTime();
        break;
    case ChartMode::FftSpectrum:
        drawFftBands();
        break;
    case ChartMode::BandPower:
        drawPsdBands();
        break;
    }
}
```

### 4.1 原始时域绘图（沿用你现有逻辑）

```cpp
void MainWindow::drawRawTime()
{
    const int n = m_plotCache.size();
    if (n <= 0) return;

    QVector<double> x(n), y(n);
    for (int i = 0; i < n; ++i) { x[i] = i; y[i] = m_plotCache[i]; }

    ensureGraphCount(1);
    m_customPlot->graph(0)->setData(x, y, true);
    m_customPlot->xAxis->setLabel(QStringLiteral("Sample"));
    m_customPlot->yAxis->setLabel(QStringLiteral("uV"));
    m_customPlot->xAxis->setRange(qMax(0, n - m_plotCacheMax), qMax(m_plotCacheMax, n));
    m_customPlot->replot(QCustomPlot::rpQueuedReplot);
}
```

### 4.2 FFT 五频段绘图（趋势线）

```cpp
void MainWindow::drawFftBands()
{
    const int n = m_fftCache.size();
    if (n <= 0) return;

    QVector<double> x(n), d(n), t(n), a(n), b(n), g(n);
    for (int i = 0; i < n; ++i) {
        x[i] = i;
        d[i] = m_fftCache[i].delta;
        t[i] = m_fftCache[i].theta;
        a[i] = m_fftCache[i].alpha;
        b[i] = m_fftCache[i].beta;
        g[i] = m_fftCache[i].gamma;
    }

    ensureGraphCount(5);
    m_customPlot->graph(0)->setData(x, d, true);
    m_customPlot->graph(1)->setData(x, t, true);
    m_customPlot->graph(2)->setData(x, a, true);
    m_customPlot->graph(3)->setData(x, b, true);
    m_customPlot->graph(4)->setData(x, g, true);
    m_customPlot->xAxis->setLabel(QStringLiteral("Window Index"));
    m_customPlot->yAxis->setLabel(QStringLiteral("FFT Amp"));
    m_customPlot->xAxis->setRange(qMax(0, n - m_featureCacheMax), qMax(m_featureCacheMax, n));
    m_customPlot->replot(QCustomPlot::rpQueuedReplot);
}
```

### 4.3 PSD 五频段绘图（趋势线）

```cpp
void MainWindow::drawPsdBands()
{
    const int n = m_psdCache.size();
    if (n <= 0) return;

    QVector<double> x(n), d(n), t(n), a(n), b(n), g(n);
    for (int i = 0; i < n; ++i) {
        x[i] = i;
        d[i] = m_psdCache[i].delta;
        t[i] = m_psdCache[i].theta;
        a[i] = m_psdCache[i].alpha;
        b[i] = m_psdCache[i].beta;
        g[i] = m_psdCache[i].gamma;
    }

    ensureGraphCount(5);
    m_customPlot->graph(0)->setData(x, d, true);
    m_customPlot->graph(1)->setData(x, t, true);
    m_customPlot->graph(2)->setData(x, a, true);
    m_customPlot->graph(3)->setData(x, b, true);
    m_customPlot->graph(4)->setData(x, g, true);
    m_customPlot->xAxis->setLabel(QStringLiteral("Window Index"));
    m_customPlot->yAxis->setLabel(QStringLiteral("Band Power"));
    m_customPlot->xAxis->setRange(qMax(0, n - m_featureCacheMax), qMax(m_featureCacheMax, n));
    m_customPlot->replot(QCustomPlot::rpQueuedReplot);
}
```

> 备注：`ensureGraphCount(5)` 的作用是确保 `QCustomPlot` 有足够 graph 对象，不够就补齐；切换回时域时可只显示第 0 条曲线，或清理多余图层。

---

## 5) 停止/清除时缓存同步策略

建议在停止和清除按钮里同时清三套缓存：

```cpp
m_plotCache.clear();
m_fftCache.clear();
m_psdCache.clear();
m_plotDirty = true;
```

这样切换图表时不会看到“上一会话残留”。

---

## 6) 实施顺序（建议）

1. 先加枚举 + 三套缓存 + 三选一弹窗（不改原绘图逻辑）
2. 再接 `fftResultReady/spectrumReady` 入缓存
3. 最后改 `onPlotRefreshTick()` 按模式分发绘图

这样每步都能单独验证，不容易一次改崩。
