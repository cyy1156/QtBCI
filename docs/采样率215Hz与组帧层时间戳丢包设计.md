# 采样率 215 Hz、组帧层时间戳与丢包观测 — 设计说明

本文档描述：**去掉在线采样率估计**、**固定按设备标称 215 Hz 处理**、**将时间戳与丢包语义前移到组帧层**，并将丢包相关信息写入 **操作日志 TXT** 与 **UI 日志区** 的改造目标与实现要点。对应代码基线为当前 QtBCI 三线程链路（采集 → 算法 → CSV/绘图）。

| 组帧时刻时间戳：`frameReady` → `lastFrameWallMs` → `RawPacket::wallMs/tsMs` | 已实现 |
| 预处理 CSV：`PlotChunk` 带 `anchorWallMs/anchorSeq`，按 215Hz 回推每点墙钟 | 已实现 |
| `SerialEegFrameAssembler`：校验失败 / 长度重同步增量信号 | 已实现 |
| 序号缺口：`RawPacket::gapSincePrev`、`streamGapDetected`、CSV 列 `gapSincePrev` | 已实现 |
| 链路诊断：`linkDiagnostic` → UI + 操作 TXT；RX 溢出沿用 `rxBufferOverflowed` | 已实现 |
| LogBuffer 溢出：`[LOG][DROP]`（与链路丢包区分） | 已实现 |

**`gapSincePrev` 与 `checksum_fail` 的区别**：`checksum_fail` 表示某一帧在校验阶段被丢弃，**不会产生**对应的 `RawPacket`；本地序号只在「成功解析并发出的样本」上递增，因此坏帧被扔掉后，下一有效样本的序号往往仍是连续递增的，`gapSincePrev` 仍为 0。只有当你能看到 **序号间断**（例如上一包 seq=10、下一包 seq=13）时，才会在同一行的 `gapSincePrev` 填缺口数量；这与是否为第一条无关（第一条同样为连续起点，缺口为 0）。

## 1. 背景与问题

### 1.1 当前数据流（摘要）

1. **串口** → `SerialEegFrameAssembler::onBytes` 组出完整帧 → `SerialEegPayloadParser::parseFrame`。
2. 解析得到 raw / signal quality 等 → `RawToUvProcessor` 产生序号与 µV → `AcquisitionEngine::onUvReady` 组装 `RawPacket` 并 `emit rawPacketReady`。
3. **时间戳**：目前在 `AcquisitionEngine::onUvReady` 用 `QDateTime::currentMSecsSinceEpoch()` 写入 `RawPacket::wallMs` / `tsMs`（即 **UV 换算完成后** 才打时间戳，而非「帧刚从字节流里闭合」的时刻）。
4. **预处理采样率**：`AlgorithmEngine::onRawPacket` 将 `pkt.seq` / `pkt.wallMs` 作为 `EegPreprocessPipeline::FsHint`，`eeg_preprocess_pipeline` 内根据相邻样本的 Δseq、Δt **在线估计** `fs_used` / `fs_inst_hz`，并通过 `pipelineFsInstReady` 传到 `CsvLogWorker`，CSV 中写入 `fsInstAlgoHz` 及按墙钟整秒统计的 `ptsIn1sUtcCompleted`。

### 1.2 用户侧痛点

- **采样率显示/用于算法的不稳定**：在线估计会随 OS 调度、USB 抖动、偶发延迟而波动，与设备真实 **215 Hz** 不一致。
- **时间戳语义不理想**：希望时间戳表示「该样本对应的数据在链路中可信对齐的时刻」，且与 **组帧** 同一抽象层，而不是紧贴在 CSV 合并或 UV 输出处临时取墙钟。
- **丢包不可见**：链路异常（校验失败、重同步、缓冲溢出）或 **序号跳跃** 应被记录，并在界面与操作 TXT 中可追踪。

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| **固定采样率 215 Hz** | 预处理、PSD/FFT 等使用 **标称 fs = 215**（或配置常量），**不再**用墙钟 + seq 做在线 Hz 估计驱动滤波/频谱的关键参数。 |
| **去掉「在线采样率」产物** | 停止依赖 `pipelineFsInstReady` → CSV `fsInstAlgoHz`；可删除该列或固定写入 `215.000`；`ptsIn1sUtcCompleted` 若仍基于墙钟则与「固定 fs」语义冲突，建议改为 **按 seq 的整 215 点计数** 或一并移除/替换。 |
| **组帧层时间戳** | 在每帧 **校验通过且即将交付解析** 的时刻记录 **高精度时间**（建议：`std::chrono::steady_clock` 单调时钟用于间隔，`system_clock` 或 `QDateTime` 用于可读 UTC；或统一 wall ms + 文档约定）。该时间沿链路传递到 `RawPacket`，CSV 仅 **透传**，不再在 UI 线程为 preproc 批量重写当前时刻。 |
| **组帧层丢包信息** | 汇总两类信息：(A) **帧级**：校验失败、长度重同步、RX 缓冲溢出（`SerialEegFrameAssembler` 已有计数器）；(B) **样本级**：`RawPacket::seq` 相对上一包的 **缺口**（期望连续递增时的丢包帧数）。 |
| **操作日志 TXT + UI** | 丢包/异常以结构化短行写入 `MainWindow::appendUiActionLog`，同时 `appendLogLine` 显示在右下角日志区；可选：周期性摘要（如每秒 `[LINK] frames=215 checksum_fail=0 gap_samples=0`）。 |

## 3. 组帧层职责细化

### 3.1 推荐挂载点

在 **`SerialEegFrameAssembler`**（或紧贴其 `frameReady` 信号的薄适配层）完成：

1. **时间戳**：`emit frameReady(QByteArray)` **之前**，为该帧绑定 `qint64 frameWallMs`（或 `struct FrameMeta { QByteArray payload; qint64 rxWallMs; quint64 frameIndex; }`）。
2. **帧级异常**：对已有 `checksumFailures()`、`lengthResyncs()`、`bufferOverflows()` 在 **增量变化** 时发信号（例如 `linkDiagnostics(quint64 checksumDelta, ...)`），避免 UI 轮询。

若暂不改动 Assembler 的 signal 签名，可采用：

- 新增 `frameReadyWithMeta(const QByteArray &, qint64 wallMs)`，或  
- 在 `AcquisitionEngine` 内对 `frameReady` 做 **queued 直连槽**：先 `wallMs = now`，再调用 `parseFrame`。

**注意**：同一物理帧只应打一个时间戳；帧内若解析出多个 raw 更新，需约定 **是否共用一个 rx 时刻** 还是按协议内子包再细分（取决于设备一帧是否对应一样本）。

### 3.2 与解析层、UV 层的衔接

- `SerialEegPayloadParser` 若保持「只收 QByteArray」，则 **帧级 wallMs** 由 `AcquisitionEngine` 在 `frameReady` 槽中缓存为 `m_lastFrameWallMs`，在随后触发的 `onRawInt16` / `onUvReady` 中填入 `RawPacket`（若一帧触发多次 raw，需确认是否与样本文档一致：通常 **最后一次或第一次** 与帧对齐，须在注释与文档中写死）。
- 更干净的做法：解析结果结构体携带 **该样本对应的 frameWallMs**，避免成员变量竞态（单线程采集线程内顺序执行时，成员变量通常也可接受）。

## 4. 固定 215 Hz 与移除在线估计

### 4.1 配置层

- 在 `EegPreprocessPipeline::Options` 将 `fs_nominal`（及用于滤波的内部 `m_fs_smooth` 初值）设为 **215.0**。
- **关闭或绕过** `updateFsHint` 中对 `fs_inst` 的平滑更新路径，使 `Result::fs_used` **恒为 215**（或在 `process()` 入口强制 `fs_used = m_opt.fs_nominal`）。

### 4.2 算法与 CSV

- `AlgorithmEngine`：**不再** `emit pipelineFsInstReady(res.fs_inst_hz)`，或改为 emit 固定值（仅为兼容过渡）。
- `CsvLogWorker`：表头中 `fsInstAlgoHz` 列改为固定 `215`、删除该列、或改名为 `fsNominalHz` 仅作记录；`advanceUtcSecondStats` 若用于「每秒点数统计」，应改为 **基于 seq 差分** 的统计（例如每秒边界内 `max(seq)-min(seq)+1`），否则墙钟秒与 215 Hz 不对齐会产生误导。

### 4.3 预处理 LogItem 时间戳修正

当前 `MainWindow` 在 `plotChunkReady` 内对整块预处理样本使用 **同一** `QDateTime::currentMSecsSinceEpoch()`，会造成 **整窗样本共享一个错误时间戳**。在组帧层统一时间戳并 **按 seq 线性插值** 或 **每样本递增 1/215 s**（基于首样本 wallMs）可改善；更简单做法：**raw 与 preproc 共用 raw 侧已写入的 `tsMs/wallMs`**（在合并层按 seq 对齐，已有 `CsvLogWorker::mergeItem` 优先 raw 的时间戳）。

## 5. 丢包定义与上报内容

### 5.1 帧级（链路质量）

利用 / 扩展 `SerialEegFrameAssembler` 统计：

- `checksumFailures` 增量 → 「校验失败次数」  
- `lengthResyncs` 增量 → 「长度重同步次数」  
- `bufferOverflows` / `rxBufferOverflowed` → 「接收缓冲溢出」  

日志示例：

```text
[2026-05-07 12:00:01.234][LINK][WARN] checksum_fail +1 total=3
[2026-05-07 12:00:05.100][LINK][ERROR] rx_buffer_overflow prevSize=65536
```

### 5.2 样本级（序号缺口）

在 `AcquisitionEngine`（或专用 `StreamGapDetector`）维护 `lastSeq`：

- 若 `pkt.seq > lastSeq + 1`，则 `gap = pkt.seq - lastSeq - 1`，表示 **缺失样本计数**（在确认 `sampleCount()` 单调且步长为 1 的前提下）。
- 日志示例：`[STREAM][WARN] seq_gap expected=1025 got=1028 missed=2`

### 5.3 写入路径

- **UI**：`MainWindow::appendLogLine(...)`（右下角 `QPlainTextEdit`）。
- **操作 TXT**：`appendUiActionLog("LINK", message)` / `appendUiActionLog("STREAM", message)`（需 `m_uiTxtLoggingEnabled` 为真且有路径）。

可选：**状态栏**短时展示最近一次 gap（`statusBar()->showMessage`），避免刷屏。

## 6. CSV 字段建议（与现有表头演进）

当前原始 CSV 表头（节选）：`tsMs,seq,...,ptsIn1sUtcCompleted,fsInstAlgoHz`。

建议演进方向（择一）：

1. **最小改动**：保留 `tsMs,seq`，`fsInstAlgoHz` 固定 215；`ptsIn1sUtcCompleted` 改为基于 seq 的计数或删除。  
2. **表达丢包**：增加可选列 `gapSincePrev`（0 表示连续，>0 表示缺失样本数）、`frameChecksumOk`（若可关联）。  
3. **文档化**：在 README 或本文档中说明 **tsMs 为组帧接收时刻**（或样本理论时刻），避免与「CSV 写入时刻」混淆。

---

## 7. 实现顺序建议

1. **固定 fs=215**：改 `EegPreprocessPipeline` + 断开/固定 `pipelineFsInstReady` + 调整 CSV 列语义。  
2. **组帧时间戳**：`frameReady` 路径挂 wallMs，贯通至 `RawPacket`，删除/合并 UI 线程对 preproc 的错误批量时间戳。  
3. **丢包信号**：Assembler 增量诊断 + seq 缺口检测 → `appendLogLine` + `appendUiActionLog`。  
4. **联调**：对比设备文档确认 **一帧一样本** 还是 **一帧多样本**，校正「帧时间戳 → 样本时间戳」映射。  
5. **回归**：无采集时无崩溃；停止采集后计数复位策略与 `resetSequence(0)` 一致。

## 8. 风险与注意事项

- **墙钟 vs 单调时钟**：丢包间隔统计应用单调时钟；用户可读 `tsMs` 用 UTC 本地格式化即可。  
- **多 raw / 帧**：若一帧解析触发多次 `onRaw`，必须为每次发射定义时间戳（共用帧时刻或子序号），否则 CSV 仍会出现时间戳扭曲。  
- **seq 复位**：设备重连后 seq 若归零，应检测 **大幅回绕** 并重置 `lastSeq`，避免误报巨大 gap。  
- **LogBuffer 丢样本**：`LogBuffer::droppedCount` 属于 **软件队列溢出**，与射频/USB **丢包** 不同，日志类别应区分（如 `[LOG][DROP]`）。
