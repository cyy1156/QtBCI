# 会话重置与逻辑修复（代码示例）

你希望实现的流程是：

1. 停止采集（或暂停）  
2. 清空算法缓冲  
3. 重置 `RawtOutUvProcessor::m_seq`  
4. 再开始采集  

下面给出一套和你当前三线程架构兼容的“会话级重置”示例（只给示例，不直接改项目）。

---

## 1) 先统一 seq 的起点语义

你当前 `RawtOutUvProcessor::buildSample()` 里是：

```cpp
s.seq = ++m_seq;
```

这意味着：
- `m_seq=0` 时第一条是 `1`。

如果你希望“第一条就是 0”，改成：

```cpp
s.seq = m_seq++;
```

---

## 2) 给 RawtOutUvProcessor 增加重置接口

文件：`thinkgear/rawtouvprocessor.h`

```cpp
public:
    void resetSequence(quint64 start = 0) { m_seq = start; }
```

---

## 3) 采集线程启动前重置 seq（并清采集状态）

文件：`core/acquisitionengine.cpp`，在 `AcquisitionEngine::start(...)` 开头：

```cpp
// 会话重置：从 0 重新开始（若你保持 ++m_seq，可传 0 让第一条为1）
m_converter->resetSequence(0);
m_lastRawInt16 = 0;
m_lastSignalQuality = 255;
```

> 你当前是 `resetSequence(1)`，如果同时还用 `++m_seq`，第一条会变成 2，不符合“从 0/1 清晰开始”的预期。

## 4) 给算法线程增加 resetState()（清窗口缓存 + 采样率状态）

文件：`core/algorithmengine.h`

public slots:

void AlgorithmEngine::resetState()
{

## 5) MainWindow 增加“会话重置并重启”的统一流程

建议新增一个辅助函数（示例名：`restartSessionWithReset`）：

文件：`mainwindow.cpp`

```cpp
void MainWindow::restartSessionWithReset(const QString &port, qint32 baud)
{
    if (!m_acq || !m_alg || !m_csvWorker)
        return;

    // 1) 停止采集与写盘
    QMetaObject::invokeMethod(m_acq, "stop", Qt::QueuedConnection);
    QMetaObject::invokeMethod(m_csvWorker, "stop", Qt::QueuedConnection);

    // 2) 清算法状态（必须在算法线程执行）
    QMetaObject::invokeMethod(m_alg, "resetState", Qt::QueuedConnection);

    // 3) 清 UI 缓存（主线程）
    m_plotCache.clear();
    m_plotDirty = false;
    if (m_customPlot && m_customPlot->graphCount() > 0) {
        m_customPlot->graph(0)->data()->clear();
        m_customPlot->replot(QCustomPlot::rpQueuedReplot);
    }

    // 4) 重新配置写盘路径（主CSV + PSD CSV）
    if (m_csvLoggingEnable && m_eegCsvPath.isEmpty())
        m_eegCsvPath = makeDefaultEegCsvPath();
    if (m_psdLoggingEnable && m_psdCsvPath.isEmpty())
        m_psdCsvPath = makeDefaultPsdCsvPath();

    QMetaObject::invokeMethod(m_csvWorker, [this]() {
        if (m_csvLoggingEnable) {
            m_csvWorker->setOutputPath(m_eegCsvPath); // 主 CSV
            m_csvWorker->start();
        }
        if (m_psdLoggingEnable) {
            m_csvWorker->setSpectrumOutputPath(m_psdCsvPath); // PSD CSV
        } else {
            m_csvWorker->setSpectrumOutputPath(QString());
        }
    }, Qt::QueuedConnection);

    // 5) 再启动采集（AcquisitionEngine::start 内会 resetSequence）
    QMetaObject::invokeMethod(m_acq, [this, port, baud]() {
        m_acq->start(port.trimmed(), baud);
    }, Qt::QueuedConnection);
}
```

## 6) 清除按钮升级为“会话重置”示例

你当前 `on_pushButton_clear_clicked()` 只是清屏，不会重置 seq。  
可以改成两种模式之一：

### 模式 A：仅清屏（保留现有）
- 快速清 UI，不影响数据流。

### 模式 B：清屏 + 会话重置（推荐新增按钮或二次确认）

```cpp
void MainWindow::on_pushButton_clear_clicked()
{
    const auto ret = QMessageBox::question(
        this,
        QStringLiteral("确认"),
        QStringLiteral("清除并重置会话？这会停止并重新开始采集，seq 从 0 重新计数。"));
    if (ret != QMessageBox::Yes)
        return;

    // 这里你可保存上次端口，或弹窗重新选择
    const QString port = QStringLiteral("COM7");
    restartSessionWithReset(port, 57600);

    appendUiActionLog(QStringLiteral("UI"), QStringLiteral("点击清除并重置会话"));
}
```

## 7) 我在你当前代码里发现的可一并修复项

### 7.1 PSD 打开按钮非 Windows 分支路径写错

你现在是：

```cpp
QDesktopServices::openUrl(QUrl::fromLocalFile(m_PsdCsvPath));
```

注意变量名大小写，建议统一成：

```cpp
QDesktopServices::openUrl(QUrl::fromLocalFile(m_psdCsvPath));
```

### 7.2 主 CSV 与 PSD CSV 路径不要互相覆盖

正确做法（你现在已基本改对）：
- 主 CSV：`setOutputPath(m_eegCsvPath)`
- PSD CSV：`setSpectrumOutputPath(m_psdCsvPath)`

不要再出现：

```cpp
setOutputPath(m_psdCsvPath); // 这会把主 CSV 写到 PSD 文件
```

### 7.3 Spectrum 类型头依赖建议（结构优化）

你现在在 `csvlogworker.h` 里 include 了 `psdfeatureextractor.h`，同时又前置声明 `SpectrumResult`。  
建议将 `SpectrumResult` 抽到独立头 `core/spectrumtypes.h`，让 `algorithmengine` / `csvlogworker` 都 include 这个头，减少耦合和循环依赖风险。

## 8) 验证步骤（重置成功判定）

1. 点击“清除并重置会话”后，日志出现：停止 -> 重启。  
2. 新会话第一条 `seq` 为 0（若你采用 `m_seq++`）或 1（若保持 `++m_seq`）。  
3. `eeg_*.csv` 与 `psd_*.csv` 都继续增长，且不会互相串写。  
4. 算法窗口不混入旧会话数据（重置后前几个窗口不会出现旧 seq 尾巴）。
